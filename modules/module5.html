Модуль 5: Git та GitHub — Зберігаємо історію

Професійна розробка неможлива без системи контролю версій. Це як страхова сітка для акробата — вона дозволяє робити сміливі кроки, знаючи, що завжди можна повернутись у безпечну точку. Наш вибір — Git, найпопулярніша система у світі.

Що таке Git? Аналогія з "сейвами" в грі

Git — це програма на твоєму комп'ютері, яка відстежує всі зміни у файлах твого проєкту. Уяви, що твій проєкт — це гра.

  • Репозиторій (Repository): Це папка твого проєкту, для якої ти увімкнула відстеження командою `git init`. Це твоя "папка з грою".
  • Коміт (Commit): Це точка збереження ("сейв"). Ти робиш коміт, коли завершила логічний шматок роботи. Кожен коміт має унікальний ID та опис, щоб ти знала, що саме було збережено.

Локальний робочий цикл: "Свята трійця"

90% щоденної роботи з Git — це цикл з трьох команд:

  1. git status — "Що змінилось?". Git показує тобі нові файли або файли зі змінами.
  2. git add . — "Підготувати до збереження". Ти кажеш Git підготувати всі поточні зміни для наступного коміту.
  3. git commit -m "Опис зміни" — "Зробити збереження". Ти створюєш коміт з коротким описом того, що саме було зроблено.

Що таке GitHub? Онлайн-сховище

Якщо Git — це сама система збережень, то GitHub — це онлайн-сервіс, "хмара", де ти можеш зберігати копії своїх репозиторіїв.

  • Резервна копія: Твій код у безпеці, навіть якщо з твоїм комп'ютером щось станеться.
  • Портфоліо: Це твоя публічна візитівка, яку ти показуєш роботодавцям.
  • Командна робота: Дозволяє багатьом розробникам працювати над одним проєктом одночасно.

Процес синхронізації з GitHub

Щоб відправити свої локальні коміти на GitHub, використовується команда `git push`.


# 1. Додаємо посилання на віддалений репозиторій (робиться один раз)
git remote add origin https://github.com/Username/Repo.git

# 2. Відправляємо коміти на GitHub
git push -u origin main

Що таке "коміт" (commit) в Git?

Резервна копія всього комп'ютера.
Знімок стану файлів у певний момент часу, "точка збереження".
Коментар у коді.

Коміт — це ключове поняття в Git. Це не просто зміна, а повний знімок (snapshot) проєкту в конкретний момент часу з унікальним ID та описом.

Яка команда покаже поточний стан репозиторію: які файли змінено, які готові до коміту?

git commit
git add
git status

`git status` — це твоя "приладова панель". Цю команду використовують найчастіше, щоб зрозуміти, що відбувається в репозиторії перед тим, як робити будь-які дії.

У чому головна відмінність між Git та GitHub?

Це одне й те саме.
Git — це програма на комп'ютері, а GitHub — це веб-сайт для хостингу репозиторіїв.
Git — для програмістів, а GitHub — для дизайнерів.

Це фундаментальна відмінність. Git — це інструмент (як молоток). GitHub — це місце, де ти зберігаєш проєкти, зроблені цим інструментом (майстерня).

Що робить команда `git add .`?

Створює новий коміт з усіма змінами.
Додає всі змінені та нові файли в поточній папці до "staging area" (підготовки до коміту).
Відправляє всі зміни на GitHub.

Це другий крок у "святій трійці" Git. Команда `git add` готує зміни до збереження. Вона не робить сам "сейв" (коміт), а лише каже: "Я хочу, щоб ось ці зміни були включені в наступне збереження". Крапка (`.`) означає "всі файли в поточній директорії".

Яка команда використовується для відправки локальних комітів на віддалений репозиторій (напр., GitHub)?

git commit
git pull
git push

Команда `git push` (дослівно "штовхати") бере всі коміти, які ви зробили локально, і відправляє їх на віддалений сервер, такий як GitHub, роблячи ваші зміни доступними для інших або просто створюючи резервну копію в хмарі.
