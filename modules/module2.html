Модуль 2: Python — Наша "чарівна паличка"

Якщо Selenium — це автомобіль, то Python — це двигун, який ним керує. Нам не потрібно знати всю інженерію, але розуміти основи роботи двигуна — обов'язково. У цьому модулі ми вивчимо "абетку" Python, яка потрібна автоматизатору щодня.

1. Змінні та Типи Даних: "Коробки" для інформації

Змінна — це як підписана коробка, в яку ми кладемо інформацію. Залежно від типу інформації, Python використовує різні "коробки".

  • Рядок (string, str): Текст у лапках.
    expected_title = "Your Profile"
  • Ціле число (integer, int): Числа без дробової частини.
    items_in_cart = 5
  • Булевий тип (boolean, bool): "Так" або "Ні". Дуже важливий для перевірок.
    is_user_logged_in = True
  • Список (list, list): Впорядкований набір елементів у квадратних дужках. Можна змінювати.
    locators = ["#username", "#password", "#login-button"]
  • Словник (dictionary, dict): Набір пар "ключ:значення" у фігурних дужках. Критично важливий для API-тестування, бо ідеально відповідає формату JSON.
    user_data = {"name": "Olena", "role": "QA"}

2. Умови (if / else): Розвилки на дорозі

Дозволяють програмі приймати рішення. "Якщо умова правдива — роби це, інакше — роби щось інше". 


actual_title = driver.title # Отримали заголовок сторінки

if actual_title == "Welcome Page":
    print("✅ Перевірка успішна: ми на правильній сторінці.")
else:
    print(f"❌ Помилка! Очікували 'Welcome Page', а отримали '{actual_title}'.")

3. Цикли (for): Робимо рутину

Дозволяють повторювати дію для кожного елемента у списку. Незамінні для перевірки таблиць, списків товарів тощо. 


# Список очікуваних посилань у меню
expected_links = ["Home", "Products", "Contacts"]

# Знаходимо всі реальні посилання на сайті
actual_links = driver.find_elements(By.CSS_SELECTOR, ".menu-item")

for link_element in actual_links:
    # Перевіряємо, чи текст кожного посилання є в нашому списку очікувань
    assert link_element.text in expected_links
    print(f"Перевірено посилання: {link_element.text}")

4. Функції (def): Створюємо власні інструменти

Це основа принципу DRY (Don't Repeat Yourself - Не повторюй себе). Ми "упаковуємо" шматок коду, який часто використовується, даємо йому ім'я і можемо потім викликати його багато разів.

До: Повторення коду


# --- Тест 1 ---
print("---=== Початок тесту логіну ===---")
# ... код тесту ...
print("---=== Кінець тесту логіну ===---")

# --- Тест 2 ---
print("---=== Початок тесту реєстрації ===---")
# ... код тесту ...
print("---=== Кінець тесту реєстрації ===---")

Після: Використання функції


# Оголошуємо функцію з аргументом
def run_test(test_name, test_function):
    print(f"---=== Початок тесту: {test_name} ===---")
    test_function() # Викликаємо передану функцію
    print(f"---=== Кінець тесту: {test_name} ===---")

# Викликаємо нашу обгортку для кожного тесту
run_test("Логін", login_test_logic)
run_test("Реєстрація", registration_test_logic)

Що буде результатом виконання коду: `x = "5"`, `y = "3"`, `print(x + y)`?

"53"
8
Помилка

Оскільки і "5", і "3" знаходяться в лапках, вони є рядками (strings). Оператор `+` для рядків означає конкатенацію (склеювання), а не математичне додавання.

Який оператор використовується для перевірки рівності двох значень?

=
==
!=

Одинарний знак `=` використовується для присвоєння значення змінній (`x = 5`), тоді як подвійний `==` використовується для порівняння ("чи дорівнює `x` п'яти?").

Що таке `список` (list) в Python?

Незмінний набір елементів.
Впорядкований, змінний набір елементів у квадратних дужках.
Набір унікальних елементів у фігурних дужках.

Списки `[...]` є однією з найпоширеніших структур даних в Python. Вони впорядковані (мають індекси) і їх можна змінювати (додавати, видаляти елементи).

Що означає ключове слово `pass` в Python?

Нічого не робити; це заглушка.
Зупинити виконання програми.
Пропустити поточну ітерацію циклу.

`pass` використовується, коли синтаксично потрібен якийсь код, але вам не потрібно, щоб щось виконувалось. Наприклад, у порожній функції або класі, який ви плануєте написати пізніше.

Що виведе цей код: `my_list = ["a", "b", "c"]`, `print(my_list[0])`?

a
b
Помилка, бо індекс має бути 1.

Пояснення

В яблучко! В Python, як і в багатьох мовах програмування, індексація елементів у списках (та інших послідовностях) починається з 0. Тому перший елемент завжди має індекс `[0]`.

Для чого в коді використовується символ `#`?

Для позначення ID елемента.
Для написання однорядкового коментаря, який ігнорується програмою.
Для створення нового блоку коду.

Пояснення

Абсолютно вірно. Все, що написано в рядку після символу `#`, є коментарем. Інтерпретатор Python повністю ігнорує цей текст. Коментарі потрібні для пояснення коду іншим розробникам (або собі в майбутньому).
