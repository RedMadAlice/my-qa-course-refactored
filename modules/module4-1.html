Модуль 4.1: Практика взаємодії з елементами

Маючи знання про РОМ, зануримось у практику автоматизації конкретних веб-елементів. Тут ми розглянемо два найпоширеніші: випадаючі списки та чекбокси.

Робота з випадаючими списками (<select>)

Випадаючі списки — це елементи <select>. Для зручної роботи з ними в Selenium є спеціальний допоміжний клас Select, який значно спрощує вибір потрібних опцій.

Клас сторінки (dropdown_page.py)


from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import Select

class DropdownPage:
    DROPDOWN_LIST = (By.ID, "dropdown")

    def __init__(self, driver):
        self.driver = driver
        self.dropdown = Select(self.driver.find_element(*self.DROPDOWN_LIST))

    def select_option_by_text(self, visible_text):
        self.dropdown.select_by_visible_text(visible_text)

    def get_selected_option_text(self):
        return self.dropdown.first_selected_option.text

Тест (test_dropdown.py)


from dropdown_page import DropdownPage

# ... (код ініціалізації драйвера) ...

driver.get("http://the-internet.herokuapp.com/dropdown")
dropdown_page = DropdownPage(driver)

option_to_select = "Option 2"
dropdown_page.select_option_by_text(option_to_select)

selected_option_text = dropdown_page.get_selected_option_text()
assert selected_option_text == option_to_select

print("✅ Тест пройдено!")
driver.quit()

Робота з чекбоксами (<input type="checkbox">)

Чекбокси — це елементи, стан яких потрібно не тільки змінювати (клікати), але й перевіряти. Для цього ідеально підходить метод .is_selected(), який повертає True або False.

Клас сторінки (checkboxes_page.py)


from selenium.webdriver.common.by import By

class CheckboxesPage:
    CHECKBOX_1 = (By.CSS_SELECTOR, "form#checkboxes input:nth-of-type(1)")
    CHECKBOX_2 = (By.CSS_SELECTOR, "form#checkboxes input:nth-of-type(2)")

    def __init__(self, driver):
        self.driver = driver

    def click_checkbox_1(self):
        self.driver.find_element(*self.CHECKBOX_1).click()

    def is_checkbox_1_selected(self):
        return self.driver.find_element(*self.CHECKBOX_1).is_selected()
    
    # ... (аналогічні методи для другого чекбоксу) ...

Тест (test_checkboxes.py)


from checkboxes_page import CheckboxesPage

# ... (код ініціалізації драйвера) ...

driver.get("http://the-internet.herokuapp.com/checkboxes")
checkboxes_page = CheckboxesPage(driver)

# Перевіряємо, що чекбокс 1 НЕ вибраний
assert not checkboxes_page.is_checkbox_1_selected()

checkboxes_page.click_checkbox_1()

# Перевіряємо, що тепер чекбокс 1 вибраний
assert checkboxes_page.is_checkbox_1_selected()

print("✅ Тест пройдено!")
driver.quit()

Для чого призначений клас `Select` в Selenium?

Для вибору будь-якого елемента на сторінці.
Для зручної роботи з випадаючими списками <select>.
Для вибору дати в календарі.

Клас `Select` — це спеціалізований інструмент, створений саме для взаємодії з HTML-елементами <select>, що робить код чистішим і надійнішим.

Який метод є найнадійнішим для вибору опції з випадаючого списку?

select_by_visible_text("Текст опції")
select_by_index(1)
click()

Вибір за видимим текстом (`select_by_visible_text`) є найнадійнішим, оскільки він не залежить від порядку елементів (як `index`) чи значень `value`, які можуть бути неінформативними.

Що повертає метод `.is_selected()`?

Текст вибраного елемента.
True, якщо елемент вибраний, інакше False.
Об'єкт елемента.

Метод `.is_selected()` повертає булеве значення (boolean), що робить його ідеальним для використання в перевірках `assert`.

Як правильно перевірити, що чекбокс НЕ вибраний?

assert not element.is_selected()
assert element.is_selected() == "False"
assert element.is_not_selected()

Оскільки `.is_selected()` повертає `False`, ми використовуємо ключове слово `not` в Python, щоб перетворити `False` на `True` і зробити перевірку `assert` успішною.
