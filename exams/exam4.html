Фінальний Екзамен (Частина 4/4): Документація та Менеджмент

Перевірка знань тестової документації, життєвого циклу дефекту та вміння ефективно взаємодіяти в команді.

Який атрибут баг-репорту є найважливішим для визначення, наскільки критичний дефект для бізнесу?

Серйозність (Severity).
Пріоритет (Priority).
Статус (Status).

"Система повністю падає при спробі завантажити аватар" — це опис дефекту з високою...

...Серйозністю (Severity).
...Пріоритетом (Priority).
...Складністю (Complexity).

Який статус отримує дефект, якщо розробник не може його відтворити?

Відхилено (Rejected).
Не вдалося відтворити (Cannot Reproduce).
Відкладено (Deferred).

Що НЕ є обов'язковим атрибутом тест-кейсу?

Кроки для відтворення.
Очікуваний результат.
Ім'я автоматизатора, який його покриє.

Яка головна відмінність між Тест-кейсом та Чек-лістом?

Тест-кейс містить детальні кроки та очікувані результати, а чек-ліст — лише список перевірок "Що" тестувати.
Тест-кейси пишуть тільки для ручного тестування, а чек-лісти — для автоматизованого.
Немає різниці.

Що таке "критерії виходу" (Exit Criteria) в Тест-плані?

Список всіх знайдених дефектів.
Умови, при досягненні яких тестування вважається завершеним (напр., 95% тест-кейсів пройдено).
Дата релізу продукту.

Дефект, який буде виправлено в наступних версіях продукту, отримує статус:

Відхилено (Rejected).
Відкладено (Deferred/Postponed).
Дублікат (Duplicate).

Який документ є основою для написання тест-кейсів?

Вимоги до програмного забезпечення (Software Requirements Specification).
Звіт про попереднє тестування.
Список знайдених дефектів.

Хто зазвичай встановлює Пріоритет (Priority) для дефекту?

Тестувальник.
Менеджер продукту або бізнес-аналітик.
Розробник.

Назва баг-репорту: "Орфографічна помилка в логотипі компанії на головній сторінці". Які, найімовірніше, будуть Серйозність та Пріоритет?

Висока Серйозність, Високий Пріоритет.
Низька Серйозність (не ламає функціонал), Високий Пріоритет (впливає на імідж).
Висока Серйозність, Низький Пріоритет.
