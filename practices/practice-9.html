Практика №9: Основи Git. Створення репозиторію та перший коміт

Мета

Зрозуміти базовий робочий процес в Git. Ви навчитеся створювати локальний репозиторій, додавати файли "під нагляд" Git та зберігати їх стан, створюючи коміт.

Очікуваний результат

Створена папка на вашому комп'ютері, яка є Git-репозиторієм, з одним файлом та одним комітом в історії.

Покрокова інструкція

Ця практика виконується у вашому терміналі (командному рядку).

  1. Створення робочої папки.

    Створіть нову папку для нашого проекту та перейдіть у неї.

    
mkdir my-git-project
cd my-git-project
  2. Ініціалізація репозиторію.

    Ця команда створює в поточній папці приховану підпапку `.git`, де Git буде зберігати всю історію та налаштування. Ваша папка стала репозиторієм.

    git init

    Ви побачите повідомлення типу "Initialized empty Git repository..."

  3. Створення файлу.

    Створимо простий текстовий файл.

    echo "Hello Git!" > readme.md
  4. Перевірка статусу.

    Ця команда показує поточний стан репозиторію. Вона дуже корисна.

    git status

    Git покаже, що у вас є "Untracked files" (файли, за якими Git ще не стежить), і запропонує `readme.md`.

  5. Додавання файлу в "staging area".

    Команда `git add` готує файл до коміту (додає його до "проміжної зони").

    git add readme.md

    Якщо ви знову виконаєте `git status`, то побачите, що `readme.md` тепер у секції "Changes to be committed".

  6. Створення коміту.

    Коміт — це знімок стану ваших файлів у певний момент часу. Кожен коміт має унікальний ID та опис.

    git commit -m "Initial commit: Add readme file"

    Прапорець `-m` дозволяє додати повідомлення до коміту прямо з командного рядка. Це повідомлення дуже важливе, воно має коротко описувати, які зміни ви зробили.

  7. Перевірка історії.

    Команда `git log` покаже історію всіх комітів у вашому репозиторії.

    git log

    Ви побачите свій перший коміт з його унікальним хешем, автором та датою.

Вітаю! Ви щойно пройшли повний базовий цикл роботи з Git.
