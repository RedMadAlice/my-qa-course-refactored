Практика №10: Робота з гілками в Git

Мета

Навчитися створювати гілки (branches), переключатися між ними та зливати (merge) зміни з однієї гілки в іншу. Гілки — це ключовий інструмент для паралельної розробки та ізоляції змін.

Очікуваний результат

Ви зрозумієте, як створити окрему гілку для нової функціональності, зробити зміни в ній, а потім інтегрувати ці зміни назад в основну гілку.

Покрокова інструкція

Продовжуємо працювати в репозиторії `my-git-project` з попередньої практики.

  1. Перевірка поточної гілки.

    За замовчуванням ви знаходитесь в основній гілці, яка зазвичай називається `master` або `main`.

    git branch

    Зірочка `*` покаже вашу поточну гілку.

  2. Створення нової гілки.

    Уявімо, ми хочемо додати нову "фічу". Створимо для неї окрему гілку.

    git branch feature/add-description

    Команда `git branch` тепер покаже дві гілки, але ви все ще на `main`.

  3. Переключення на нову гілку.

    Щоб почати роботу над фічею, нам потрібно перейти в її гілку.

    git checkout feature/add-description

    (Професійна порада: команди `git branch` та `git checkout` можна об'єднати: `git checkout -b feature/add-description` одразу створить гілку і переключиться на неї).

  4. Внесення змін у новій гілці.

    Додамо новий рядок до нашого файлу `readme.md`.

    echo "This is a project for Git practice." >> readme.md

    Тепер закоммітимо ці зміни. Пам'ятайте, ми робимо це в гілці `feature`!

    
git add readme.md
git commit -m "Feat: Add project description"
  5. Повернення в основну гілку.

    Наша "фіча" готова. Повернемося в `main`, щоб злити наші зміни.

    git checkout main

    Якщо ви зараз подивитесь на вміст `readme.md` (напр., `cat readme.md`), ви побачите, що другого рядка там немає. Це тому, що зміни існують лише в гілці `feature`.

  6. Злиття (Merge) гілок.

    Тепер "втягнемо" зміни з `feature/add-description` в нашу поточну гілку (`main`).

    git merge feature/add-description

    Git автоматично об'єднає зміни. Тепер, якщо ви перевірите `readme.md`, він буде містити обидва рядки.

  7. Прибирання.

    Після того, як гілка злита і більше не потрібна, її можна видалити.

    git branch -d feature/add-description

Ви освоїли один з найважливіших процесів у Git — `feature branch workflow`. Це основа командної роботи над будь-яким проектом.
